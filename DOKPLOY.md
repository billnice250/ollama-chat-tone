# Dokploy Template Creation Guide

This guide explains how to create a Dokploy template for `Ollama Chat Tone` and add it to the Dokploy Templates repository.

## Goal

Create a Dokploy template that deploys the published Docker Hub image `billnice250/chattone:latest`.

## Where to add the template

In the Dokploy repository, add a new folder under `blueprints/`:

- `blueprints/ollama-chat-tone/`

Inside that folder, add:

1. `docker-compose.yml`
2. `template.toml`
3. `ollama-chat-tone.svg` (template logo)

Then update the root `meta.json` file with the new template metadata.

---

## 1. `docker-compose.yml`

Use a Dokploy-friendly compose file with service names and a single mapped port.

Example:

```yaml
version: "3.8"
services:
  ollama-chat-tone:
    image: billnice250/chattone:latest
    ports:
      - 8080
    environment:
      APP_NAME: "Ollama Chat Tone"
      ADDR: ":8080"
      SESSION_SECRET: "${session_secret}"
      DB_PATH: "/data/app.db"
      OLLAMA_URL: "${ollama_url}"
      OLLAMA_TIMEOUT: "${ollama_timeout}"
      DEFAULT_MODEL: "${default_model}"
      OPEN_BROWSER: "false"
      BASIC_AUTH_USER: "${basic_auth_user}"
      BASIC_AUTH_PASSWORD: "${basic_auth_password}"
    volumes:
      - ollama-chat-tone-data:/data

volumes:
  ollama-chat-tone-data: {}
```

### Notes

- Do not include `container_name`.
- Do not define custom networks.
- Use `ports: [8080]` instead of `8080:8080`.

---

## 2. `template.toml`

Define variables, service domains, environment variables, and mounts.

Example:

```toml
[variables]
main_domain = "${domain}"
session_secret = "${password:32}"
ollama_url = "http://your-ollama-host:11434"
ollama_timeout = "5"
default_model = "llama3.2"
basic_auth_user = "admin"
basic_auth_password = "${password:16}"

[config]
[[config.domains]]
serviceName = "ollama-chat-tone"
port = 8080
host = "${main_domain}"
path = "/"

[config.env]
APP_NAME = "Ollama Chat Tone"
ADDR = ":8080"
SESSION_SECRET = "${session_secret}"
DB_PATH = "/data/app.db"
OLLAMA_URL = "${ollama_url}"
OLLAMA_TIMEOUT = "${ollama_timeout}"
DEFAULT_MODEL = "${default_model}"
OPEN_BROWSER = "false"
BASIC_AUTH_USER = "${basic_auth_user}"
BASIC_AUTH_PASSWORD = "${basic_auth_password}"

[[config.mounts]]
serviceName = "ollama-chat-tone"
filePath = "/data/app.db"
content = ""
```

### Notes

- `serviceName` must match the service name in `docker-compose.yml`.
- Use Dokploy variable helpers for secrets.
- `filePath` in mounts should correspond to the runtime file path.

---

## 3. Add metadata to `meta.json`

Add a new object entry for the template metadata.

Example:

```json
{
  "id": "ollama-chat-tone",
  "name": "Ollama Chat Tone",
  "version": "latest",
  "description": "Self-contained Ollama chat client with local auth, streaming responses, and per-user conversations.",
  "logo": "ollama-chat-tone.svg",
  "links": {
    "github": "https://github.com/billnice/ollama-chat-client",
    "website": "https://github.com/billnice/ollama-chat-client"
  },
  "tags": [
    "chatbot",
    "ai",
    "ollama",
    "docker"
  ]
}
```

### Metadata requirements

- `links` must be an object with named keys.
- Use lowercase `tags`.
- Keep the entry readable and consistent with existing templates.

---

## 4. Logo

Add a template icon file to the blueprint folder:

- `ollama-chat-tone.svg`

This file should be a simple SVG with the app mark or wordmark.

---

## 5. Submit the PR

1. Fork `https://github.com/Dokploy/templates`
2. Create a branch
3. Add the blueprint folder and metadata changes
4. Commit and push
5. Open a pull request

### Checklist

- [ ] `blueprints/ollama-chat-tone/docker-compose.yml`
- [ ] `blueprints/ollama-chat-tone/template.toml`
- [ ] `blueprints/ollama-chat-tone/ollama-chat-tone.svg`
- [ ] meta.json updated
- [ ] metadata uses `links` object and lowercase `tags`
- [ ] no `container_name`
- [ ] ports configured as `8080`
- [ ] `template.toml` service names match compose service names
- [ ] tested in Dokploy preview

---

## 6. Testing

After creating the PR:

1. Use the preview URL generated by Dokploy.
2. Open the template card.
3. Copy the Base64 configuration or Base URL.
4. Import it into your Dokploy instance.
5. Deploy and verify the app starts and exposes the UI.
